# Project Principles

## Vision

Soulbound: Chronicles of Valtheris is designed as a high-quality mobile-first Action RPG that can naturally evolve into a full MMORPG.

Every new feature should support this long-term vision.

---

## Core Principles

### 1. Mobile First

The game must always be comfortable to play on mobile devices.

---

### 2. Modular Architecture

Every new gameplay system should be implemented as a separate module whenever possible.

Avoid adding large systems directly into App.tsx.

---

### 3. Performance First

The game should run smoothly even on low-end mobile devices.

---

### 4. No Pay to Win

If monetization is added in the future, it must never provide unfair gameplay advantages.

---

### 5. Dark Fantasy Style

All locations, enemies, UI, music and effects should follow the Dark Fantasy atmosphere.

---

### 6. MMORPG Ready

Every new system should be designed so it can later work in multiplayer with minimal changes.

---

### 7. Meaningful Progression

Players should always feel that their character is becoming stronger.

---

### 8. Reward Exploration

Exploring the world should always be rewarded.

Examples:
- Hidden chests
- Secret passages
- Rare enemies
- Hidden bosses
- Lore discoveries

---

### 9. Reward Skill

Player knowledge and decision making should matter more than grinding.

---

### 10. Easy to Expand

Every major feature should allow future expansion without complete rewrites.

---

## Development Workflow

1. Plan the feature.
2. Implement the feature.
3. Test the feature.
4. Update documentation.
5. Commit changes.
6. Push to GitHub.

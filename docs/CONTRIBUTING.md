# Contributing Guide

## Development Rules

When adding new features:

1. Do not rewrite existing systems.
2. Keep code modular.
3. Create separate files whenever possible.
4. Update CHANGELOG.md.
5. Update ROADMAP.md if needed.
6. Test all changes before committing.
7. Keep mobile compatibility.
8. Preserve save compatibility.

## Commit Style

Examples:

feat: add talent system

fix: equipment HP calculation

refactor: move combat into module

docs: update roadmap
